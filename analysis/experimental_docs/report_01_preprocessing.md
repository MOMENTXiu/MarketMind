# 实验报告 01 · 数据预处理

> 脚本：`data_preprocessing.py` ｜ 数据：`data/销售数据.csv` (GBK, 42816 行 × 17 列)

## 1. 目标
将原始销售明细清洗为可建模的标准化明细表，逐列识别并处理数据质量问题（思路文档 §4），产出 `cleaned_sales_data.csv` 及数据质量诊断图表。

## 2. 逐列质量诊断与处理

| 列 | 问题 | 处理 | 影响行数 |
|---|---|---|---|
| 全表 | 完全重复行 | 去重 | 3 |
| 商品类型/是否促销/销售数量 | 规格含逗号致整行右移（错位行）| 按 金额/单价 还原 | 2 |
| 销售日期 | 非闰年 `20250229` 非法 | 回填当月最后一天 | 2 |
| 销售数量/金额 | ≤0（退货）| 标记 `is_return=1`，正向建模剔除 | 93 |
| 商品单价 | =0 | 同商品→同小类→全局中位数填补 | 2 |
| 规格型号 | 空白 | →「未知规格」| 380 |
| 单位 | 59 种同义/脏值 | 统一映射至 43 种 | — |
| 是否促销 | 是/否 | →1/0 | 全部 |
| 类目编码 | — | 验证编码↔名称一一对应，以编码为主键 | — |

**单位映射核心规则**：`千克/KG/kg/Kg/公斤/散称→千克`；`袋/袋(全角)/d袋→袋`；`合→盒`、`付→副`；脏值(`2/0/160g/一般/快/装`/空白)→未知单位。详见 `output/csvs/单位映射表.csv`。

**错位行修复伪代码**：
```
for row where is_promo not in {是,否}:
    spec      ← 规格型号 + " " + 商品类型      # "牛魔空版 12g*8"
    item_type ← "一般商品"
    amount    ← 商品单价列的值 (9.9)
    unit_price← 是否促销列的值 (9.9)
    quantity  ← round(amount/unit_price, 4)
    is_promo  ← 否
```

## 3. 结果

- 清洗后：**42813 行 × 21 列**，2611 顾客，6140 商品，正向购买 42720 行（退货 93）。
- 新增派生列：`weekday / is_weekend / week_of_year / is_return`。
- 日期范围 2025-01-01 ~ 2025-04-30（4 个月，117 个销售日）。
- 促销记录占比 **15.19%**（6502 行）。

完整统计见 `output/csvs/数据质量报告.csv`。

## 5. 自评估（对照思路文档 §4）

| 验收点 | 结论 |

| 字段标准化 | ✅ 17 列全部映射为内部字段 |

| 金额≤0 异常处理 | ✅ 标记 `is_return`，保留供分析、建模剔除 |
| 单价缺失填补 | ✅ 同商品→同小类中位数 |
| 促销二值化 | ✅ |
