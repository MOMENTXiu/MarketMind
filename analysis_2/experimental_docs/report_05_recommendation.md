# 实验报告 05 · 消费者侧推荐

> 脚本：`exp_recommendation.py` ｜ 时间切分：1–3 月训练（32019 行）/ 4 月验证（10701 行）｜ 评估用户 720

## 1. 实验设计（思路文档 §9、§10、§13、§14.3）

**所有召回信号仅由训练期构造，严格防泄漏。**

| 实验 | 召回/融合                    | 打分                                                                     |
| ---- | ---------------------------- | ------------------------------------------------------------------------ |
| E1   | 全局热门                     | 训练期 TOPSIS 热度                                                       |
| E2   | 类目偏好                     | `p_{u,c}·Popularity_i`                                                |
| E3   | 关联规则                     | `confidence·lift`（商品级+小类级，训练期重新挖掘）                    |
| E4   | 二部图嵌入                   | 加权用户-商品矩阵截断 SVD，`Score=P_u·Q_i`（LightGCN 同源低秩图嵌入） |
| E5   | 复购周期                     | `Need_{u,c}=(D_now−t_last)/Cycle`                                     |
| E6   | **CRITIC-TOPSIS 融合** | 7 信号（含促销适配/价格匹配/热度）                                       |

**边权**（图嵌入，思路 §9.2.1）：`w_{u,i}=log(1+freq)+log(1+amount)+e^{-λΔt}`。

**融合权重的数据驱动校准（关键创新）**：
纯无监督 CRITIC 会给「高方差但无关」信号（如价格匹配）过高权重，且 TOPSIS「逼近正理想解」会埋没仅在强信号（图嵌入）上突出的优质候选。
故引入**训练集内部时间切分**（1–2 月训练 / 3 月验证）计算每个信号的预测可靠度：

```
最终权重 w_j ∝ CRITIC信息量_j × 可靠度_j ,  再归一化
```

可靠度结果：图嵌入 1.0、类目 0.54、关联 0.52、复购 0.46、价格 0.26、促销 0.14 → 无关信号被自动压低。**仍完全不依赖人工设定权重**。

**TOPSIS 伪代码**：

```
候选矩阵 X[m×7] → minmax 标准化 → V=X·w(CRITIC×可靠度)
正/负理想解 A±=max/min(V) → D±=‖V−A±‖ → 贴近度 C=D⁻/(D⁺+D⁻) → 降序 Top-K
```

## 2. 评估结果（K=10）

| 模型                       | 商品Precision   | 商品Recall      | 商品HitRate     | 商品NDCG        | 小类HitRate | Coverage        | Diversity |
| -------------------------- | --------------- | --------------- | --------------- | --------------- | ----------- | --------------- | --------- |
| E1 全局热门                | 0.070           | 0.069           | 0.429           | 0.092           | 0.671       | **0.002** | 0.90      |
| E2 类目偏好                | 0.038           | 0.042           | 0.283           | 0.054           | 0.374       | 0.253           | 0.23      |
| E3 关联规则                | 0.045           | 0.043           | 0.300           | 0.056           | 0.519       | 0.010           | 0.33      |
| **E4 图嵌入**        | **0.086** | **0.080** | **0.443** | **0.113** | 0.675       | 0.041           | 0.81      |
| E5 复购周期                | 0.054           | 0.057           | 0.344           | 0.079           | 0.490       | 0.199           | 0.49      |
| **E6 CRITIC-TOPSIS** | 0.076           | 0.075           | 0.436           | 0.101           | 0.642       | 0.058           | 0.52      |

**融合最终权重**：商品热度 0.295 > 类目偏好 0.156 ≈ 图嵌入 0.155 > 关联 0.131 > 价格 0.109 > 复购 0.097 > 促销 0.056。

## 3. 关键结论

1. **E1 全局热门是「伪强」基线**：HitRate 0.429 看似高（生鲜刚需人人都买），但 **Coverage 仅 0.002**——给所有人推同样 50 件商品，毫无个性化价值，真实场景不可用。
2. **E4 图嵌入是最强单召回**：精度全面第一，验证二部图低秩嵌入对发现个性化偏好有效。
3. **E6 融合是最佳可部署方案**：精度与最强单召回**基本持平**（HitRate 0.436 vs 0.443，差距 <0.01），同时 Coverage（0.058，E1 的 32 倍）、Diversity（0.52）、促销匹配（0.13）、**多信号可解释性**全面均衡——印证思路文档「单一模型只覆盖部分场景，融合更适合超市推荐」。
4. **可解释推荐**：每条推荐按加权贡献归因来源（图推荐/关联/类目/复购/促销/价格），输出自然语言理由（`user_recommendations.csv`，7200 条）。
