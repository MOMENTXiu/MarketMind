# 数据正则化引擎 · 实验汇总报告（全量结果）

> 模块：`code_files_2/regularization/`（9 模块）+ `run_regularization.py`
> 测试数据：`data/test_data/order_1.csv`、`order_2.csv` + 原项目 `data/销售数据.csv`
> 产物：`output_2/regularized/{数据集}/`（dataset.csv + 7 个 json）
> 设计依据：`docs/MarketMind 数据正则化模块分析与设计思路报告.md`

## 0. 目标
将任意来源零售数据（不同语言列名 / 编码 / 格式 / 字段口径）自动 **识别→映射→校验→清洗→补全** 为 MarketMind 标准 Schema，使下游分析对数据接入具有普适性。本阶段仅做正则化。

## 1. 引擎组件（处理流水线）
`FileReader（编码/Excel/表头）→ SchemaProfiler（字段画像）→ SchemaMapper（四维打分+置信度）→ TypeNormalizer（类型+时间派生）→ BusinessNormalizer（业务补全/降级）→ QualityChecker → CapabilityChecker → 落盘`

**关键设计**：字段映射打分 `0.45·名称+0.25·类型+0.20·模式+0.10·分布`；别名精确命中直给≥0.90；列名证据<0.5 封顶 0.65（杜绝蹭分错配）；**自动采用阈值 0.90**，0.70–0.90 标 need_review 待人工确认，<0.70 仅候选。
