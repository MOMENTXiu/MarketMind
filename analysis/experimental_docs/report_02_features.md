## 2. 顾客画像（2611 人 × 27 列）→ `customer_profile.csv`

| 维度 | 特征 | 计算 |
|---|---|---|
| RFM | R_最近购买间隔 | `D_max − 末次购买日` |
| | F_购买频次 | 不同购买日数 |
| | M_消费金额 | 金额累计 |
| | 客单价 / 平均件单价 | M/F；金额/数量 |
| 类目偏好 | 占比_蔬果/休闲/粮油/日配/洗化/酒饮/肉禽/熟食 | 大类金额占比 |
| 集中度 | 类目熵 | `−Σ p·ln p`（小类层级）|
| | 小类购买数 / 生鲜占比 | |
| 促销敏感 | 促销金额占比、促销频次占比 | |
| | **促销敏感度** | 二者 **CRITIC 自动加权** 合成（权重 0.541 / 0.459，非人工设定）|
| 价格敏感 | 低价带占比、高价带占比、偏好价格分位 | 基于小类内价格分位 PriceRank |
| 复购 | 复购紧迫度均值、平均复购周期 | 见复购周期表 |
| 风险 | 退货率 | 退货笔数/总笔数 |

**复购周期表** `repurchase_cycle.csv`（28249 条 user×小类）：对购买≥2 次的 (顾客,小类) 计算平均间隔 `Cycle` 与紧迫度 `Need=(D_now−t_last)/Cycle`，供复购召回使用。

**算法伪代码（价格分位与价格带）**：
```
对每个小类 c:
    item_price[i] = mean(unit_price | item i in c)
    price_rank[i] = percentile_rank(item_price[i] within c)
    band[i] = 低/中/高 by price_rank ∈ [0,1/3),[1/3,2/3),[2/3,1]
LowPriceRate_u = Σ amount(price_rank≤1/3) / M_u
```

## 3. 商品画像（6138 商品 × 20 列）→ `product_profile.csv`

字段：三级类目、类型、单位、平均单价、销售金额/数量/笔数、购买人数、复购率、复购率_平滑、促销占比、价格分位/价格带、综合热度、热度排名。

**综合热度 = TOPSIS(销售金额, 销售数量, 购买人数, 复购率)**，权重由 CRITIC 自动计算。

两处关键修正（自评估迭代）：
1. **复购率经验贝叶斯收缩**：原始复购率在「购买人数=1」时虚高为 1.0，导致单买家商品霸榜。改用 `(repeat+ḡ·K)/(buyers+K), K=10` 朝全局均值收缩。
2. **金额/数量/人数 log1p 压缩长尾**，避免极端大额主导 min-max。

修正后 CRITIC 权重均衡（金额 0.258 / 数量 0.202 / 人数 0.252 / 复购 0.289），热度 Top10 变为新鲜蛋品(424 人)、蕉类、其它水果、花果等真实高频生鲜，**结果可信**。
