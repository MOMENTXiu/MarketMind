# 超市AI营销系统 - 项目规划文档

> **项目性质**: 课程实践项目
> **团队规模**: 6-7人
> **开发周期**: 4-6周
> **演示时间**: 10分钟

---

## 一、项目目标

构建一个**超市AI营销系统**，实现数据驱动的营销决策支持。

### 核心功能（对应作业要求）
1. **商品促销策略制定** - 关联规则挖掘
2. **销售额与利润预测** - 时间序列预测
3. **客户聚类分析** - 客户分群与策略
4. **AI 文本建议** - 自动生成客户营销建议和商品洞察

### 技术要求
- Python开发环境
- UI界面（图形界面或命令行）
- 算法可解释性（需要讲解算法选择理由）

## 三、功能模块设计

### 3.1 模块一：关联规则挖掘

**输入**: 订单数据（订单ID、商品列表）
**处理流程**:
```
订单数据 → 构建购物篮 → Apriori算法 → 提取后项规则 → 生成策略
```
**输出**:
- Top 10 关联规则表格
- 促销策略建议
- 可视化图表（支持度-置信度散点图、提升度条形图）

**算法参数**:
- 最小支持度: 0.02（2%）
- 最小置信度: 0.3（30%）
- 最小提升度: 1.0

**选择理由**:
- Apriori算法是购物篮分析的经典算法
- 参数通过数据分布调整，确保规则数量适中且有意义

---

### 3.2 模块二：销售预测

**输入**: 历史销售数据（日期、销售额、利润）
**处理流程**:
```
原始数据 → 周度聚合 → 特征工程(17个特征) → 模型训练 → 未来预测
```
**输出**:
- 下季度（13周）销售额预测
- 下季度利润预测
- 预测曲线可视化
- 模型性能指标（R²、RMSE、MAE）

**特征工程**:
| 类型 | 特征 | 数量 |
|------|------|------|
| 时间特征 | 周序号、月份、季度 | 3 |
| 周期特征 | sin/cos编码 | 4 |
| 滞后特征 | lag1~lag4 | 4 |
| 滑动统计 | 移动平均、标准差 | 4 |
| 差分特征 | diff、pct_change | 2 |

**算法选择**: 岭回归（Ridge Regression）
- **理由1**: 样本量有限（212周），岭回归防止过拟合
- **理由2**: 可解释性强，系数代表特征重要性
- **理由3**: R²达到0.91/0.92，满足业务需求

---

### 3.3 模块三：客户聚类

**输入**: 客户数据（客户ID、订单历史）
**处理流程**:
```
客户数据 → RFM特征计算 → 标准化 → K-Means聚类 → 分群画像 → 策略制定
```
**输出**:
- 4个客户群体画像
- 各群体营销策略
- RFM对比图、分布散点图

**RFM模型**:
- **R** (Recency): 最近购买距今天数
- **F** (Frequency): 购买频次
- **M** (Monetary): 消费金额

**聚类数选择**: K=4
- **方法**: 肘部法则 + 轮廓系数
- **理由**: 轮廓系数在K=4时最高（0.265），分群边界清晰

**客户分群**:
| 群体 | 特征 | 营销策略 |
|------|------|----------|
| 高价值活跃 | 高F高M低R | VIP专属优惠、积分加倍 |
| 普通活跃 | 中F中M中R | 满减券、交叉销售 |
| 低价值流失 | 低F低M高R | 大额满减、秒杀活动 |
| 高价值流失预警 | 低F高M高R | 召回券、专属折扣 |

---

### 3.4 模块四：完整报告与文本建议

**输入**: 三个模块的分析结果、客户画像或商品关联数据
**处理流程**:
```
分析结果 → 报告生成 / LLM 文本建议 → Markdown 或前端文本展示
```
**输出**:
- Markdown 报告文件
- 客户营销建议文本

**技术选择**: LLM Provider
- **理由**: 保留 AI 文本能力，避免语音合成依赖和音频文件生命周期

## 八、关键算法说明（用于演示讲解）

### 为什么选择Apriori算法？
- **业务匹配**: 购物篮分析的标准算法
- **可解释性**: 规则清晰，易于理解
- **参数可控**: 支持度/置信度/提升度调整

### 为什么选择岭回归？
- **样本量考虑**: 212周数据，岭回归防止过拟合
- **稳定性**: L2正则化，模型鲁棒
- **性能**: R²=0.91，满足预测精度要求

### 为什么选择K-Means？
- **效率高**: 计算快，适合客户分群
- **可解释**: 聚类中心代表典型客户
- **RFM适配**: RFM特征适合欧式距离
