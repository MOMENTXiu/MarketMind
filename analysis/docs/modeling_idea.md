# 超市营销推荐系统模型文档

> **修订说明（v2，依据实验结果回填）**：本版在原始方案基础上，根据 42816 条真实销售数据的实验结论对部分模型做了数据驱动的调整，主要变化：
> 1. **顾客分群**：实验发现零售顾客行为是「双模态连续谱」而非离散簇，UMAP+HDBSCAN 只能稳定给出 2 个自然宏观群。故落地方案改为 **AutoEncoder+GMM 软分群（7 营销群体）为主、UMAP+HDBSCAN 验证自然结构为辅**（指标见 §7、§14.1）。
> 2. **图推荐召回**：以**加权用户-商品二部图的截断 SVD 低秩嵌入**实现（与 LightGCN 同源），轻量稳健（§9.2.1）。
> 3. **CRITIC-TOPSIS 融合**：增加**训练集内部时间切分校准信号可靠度**，自动压低无关高方差信号，仍不依赖人工权重（§10）。
> 4. **商品热度**：对销量类指标 log 压缩、对复购率做经验贝叶斯收缩，修正小样本虚高（§6.2）。
> 5. **新增因果推断**：促销响应分析引入**双重机器学习（DML）**估计去偏因果效应，纠正朴素 PromoLift 的混淆偏差（§11.3、创新点八）。

## 1. 项目定位

本项目基于超市销售明细数据，构建一个面向双端用户的智能营销推荐系统。数据共有 **42,816 条销售记录、2,612 个顾客编号**，包含三级商品类目、商品编码、商品属性、销售数量、销售金额、商品单价、促销状态等信息。

本项目的核心目标不是单纯做销售统计，而是围绕“消费者买什么、为什么买、还能推荐什么；营销者应该推什么、推给谁、如何组合促销”构建一套可解释、可落地、具有一定创新性的推荐与营销决策模型。

系统面向两类用户：

| 用户类型   | 关注问题                                                     | 模型输出                                         |
| ---------- | ------------------------------------------------------------ | ------------------------------------------------ |
| 消费者用户 | 我可能需要买什么？哪些商品适合我？哪些促销值得关注？         | 个性化商品推荐、复购提醒、促销商品推荐、推荐理由 |
| 营销者用户 | 哪些顾客值得重点运营？哪些商品适合组合促销？哪些类目有增长机会？ | 顾客分群、商品组合策略、促销策略、品类经营建议   |

因此，本项目采用双侧建模框架：

```text
销售明细数据
    ↓
数据清洗与特征工程
    ↓
顾客行为画像 + 商品层级画像
    ↓
消费者侧推荐模型
    ↓
营销者侧决策模型
    ↓
个性化推荐清单 + 营销策略报告
```

# 3. 总体技术路线

本项目采用：

> **层级商品画像 + 顾客嵌入画像 + 深度嵌入软分群(AE+GMM)/密度结构验证(UMAP-HDBSCAN) + 高效关联规则(FP-Growth+HUIM) + 二部图嵌入推荐 + CRITIC-TOPSIS 自适应排序 + DML 因果推断**

整体流程如下：

```text
原始销售明细数据
    ↓
字段标准化、异常检测、时间特征构建
    ↓
顾客画像构建：
RFM + 类目偏好 + 促销敏感度 + 价格敏感度 + 复购周期
    ↓
商品画像构建：
三级类目 + 价格带 + 商品类型 + 销售热度 + 促销属性
    ↓
顾客行为嵌入：
用户-商品二部图 / 用户-类目图 / 共购图
    ↓
消费者侧模型：
图推荐 + 类目偏好推荐 + 复购推荐 + 促销适配推荐
    ↓
营销者侧模型：
密度顾客分群 + 高效关联规则 + 高价值商品组合 + 促销响应分析
    ↓
多指标融合排序：
CRITIC / TOPSIS / Pearson 去冗余
    ↓
输出：
消费者推荐清单 + 营销者策略报告 + 可视化图表
```

## 4.2 异常检测

销售数量存在小数是合理现象，因为蔬菜、水果、肉类等商品可能按重量销售。因此不能将销售数量强制转为整数。

建议使用两级异常处理。

### 4.2.1 规则型异常处理

| 字段     | 处理方式                         |
| -------- | -------------------------------- |
| 销售日期 | 转为 datetime                    |
| 销售数量 | 转为 float，允许小数             |
| 销售金额 | 转为 float，金额 ≤ 0 标记为异常  |
| 商品单价 | 缺失时用同商品或同小类中位数填补 |
| 是否促销 | “是”转 1，“否”转 0               |
| 商品规格 | 缺失填“未知规格”                 |
| 单位     | 缺失填“未知单位”                 |

### 4.2.2 模型型异常检测

为了体现创新性，可以加入 **Isolation Forest** 或 **Local Outlier Factor** 对异常销售记录进行识别。

输入特征包括：

```text
销售数量
销售金额
商品单价
同小类单价分位数
同顾客单次消费金额
是否促销
```

异常检测结果不一定直接删除，而是分为：

| 异常类型           | 处理方式                 |
| ------------------ | ------------------------ |
| 金额明显错误       | 剔除                     |
| 数量异常但金额合理 | 保留并标记               |
| 高金额大额采购     | 保留，作为高价值消费行为 |
| 促销导致低价异常   | 保留，作为促销行为       |

# 5. 顾客画像模型

顾客画像是消费者推荐和营销者分群的共同基础。

## 5.1 RFM 行为特征

对顾客 (u)，定义最近购买间隔：
$$
R_u=D_{\max}-D_u^{last}
$$
购买频次：
$$
F_u=|\{d\mid 顾客u在日期d有购买记录\}|
$$
累计消费金额：
$$
M_u=\sum_i amount_{u,i}
$$
客单价：
$$
AOV_u=\frac{M_u}{F_u}
$$
其中：

- (R_u) 越小，说明越活跃；
- (F_u) 越大，说明购买频次越高；
- (M_u) 越大，说明消费贡献越高；
- (AOV_u) 越大，说明单次购买价值越高。

## 5.2 类目偏好特征

对顾客 (u) 在类目 (c) 上的消费占比：
$$
p_{u,c}=
\frac{amount_{u,c}}{\sum_c amount_{u,c}}
$$
分别计算：

- 大类偏好；
- 中类偏好；
- 小类偏好。

例如：

| 顾客   | 蔬果占比 | 粮油占比 | 日配占比 | 休闲占比 |
| ------ | -------- | -------- | -------- | -------- |
| 用户 A | 0.48     | 0.19     | 0.21     | 0.12     |
| 用户 B | 0.15     | 0.54     | 0.18     | 0.13     |

用户 A 更偏生鲜蔬果，用户 B 更偏粮油刚需。

## 5.3 类目集中度

用类目熵衡量顾客偏好是否集中：
$$
H_u=-\sum_c p_{u,c}\log(p_{u,c})
$$
解释：

- (H_u) 高：购买类目丰富，适合跨品类推荐；
- (H_u) 低：偏好集中，适合同类升级、替代商品推荐。

## 5.4 促销敏感度

顾客促销金额占比：
$$
PromoAmountRate_u=
\frac{
\sum_i amount_{u,i}I(promo_i=1)
}{
\sum_i amount_{u,i}
}
$$
顾客促销购买次数占比：
$$
PromoFreqRate_u=
\frac{
\sum_i I(promo_i=1)
}{
N_u
}
$$
综合促销敏感度可定义为：
$$
PromoSensitivity_u=
\alpha PromoAmountRate_u+
(1-\alpha)PromoFreqRate_u
$$
其中 (\alpha) 不建议人工固定，可以用熵权法或 CRITIC 方法根据数据自动确定。

## 5.5 价格敏感度

为了避免不同品类价格天然不可比，应在同一小类内部计算价格带。

对商品 (i) 在其小类 (c) 中的价格分位数：
$$
PriceRank_i=
\frac{
rank(unit_price_i \mid i\in c)
}{
|c|
}
$$
顾客低价偏好：
$$
LowPriceRate_u=
\frac{
\sum_i amount_{u,i}I(PriceRank_i\leq 0.33)
}{
\sum_i amount_{u,i}
}
$$
高价偏好：
$$
HighPriceRate_u=
\frac{
\sum_i amount_{u,i}I(PriceRank_i\geq 0.67)
}{
\sum_i amount_{u,i}
}
$$
如果某顾客低价偏好和促销敏感度都高，则可定义为“价格敏感型消费者”。

## 5.6 复购周期特征

超市商品具有明显复购属性，例如生鲜、粮油、日配、纸品等。

对顾客 (u) 和小类 (c)，定义购买日期序列：
$$
T_{u,c}=\{t_1,t_2,\cdots,t_n\}
$$
平均复购周期：
$$
Cycle_{u,c}=
\frac{1}{n-1}\sum_{k=2}^{n}(t_k-t_{k-1})
$$
当前复购紧迫度：
$$
Need_{u,c}=
\frac{
D_{now}-t_n
}{
Cycle_{u,c}+\epsilon
}
$$
若 (Need_{u,c}\approx 1)，说明顾客可能快到再次购买该类商品的时间。

# 6. 商品画像模型

商品画像用于消费者推荐，也用于营销者进行品类经营分析。

## 6.1 商品基础画像

对每个商品 (i)，构造：

| 特征     | 含义               |
| -------- | ------------------ |
| 所属大类 | 商品一级类目       |
| 所属中类 | 商品二级类目       |
| 所属小类 | 商品三级类目       |
| 商品类型 | 生鲜 / 一般商品等  |
| 单位     | 个、袋、千克、盒等 |
| 商品单价 | 商品价格水平       |
| 是否促销 | 是否促销商品       |
| 销售金额 | 商品销售贡献       |
| 销售数量 | 商品销量表现       |
| 购买人数 | 商品覆盖顾客数     |
| 复购率   | 商品是否有重复购买 |

## 6.2 商品热度

商品热度不能只看销量，也应考虑金额和购买人数。

定义商品综合热度：
$$
Popularity_i=
TOPSIS(
\log(1+SalesAmount_i),\ \log(1+SalesQty_i),\ \log(1+BuyerCount_i),\ \widetilde{RepurchaseRate_i}
)
$$
其中各指标权重不手动设置，而由 CRITIC 或熵权法自动计算。

> **实验修正**：销量类指标经 (\log(1+\cdot)) 压缩长尾，避免极端大额商品主导 min-max 标准化；复购率采用经验贝叶斯收缩
> $$\widetilde{RepurchaseRate_i}=\frac{repeat_i+\bar g\cdot K}{buyers_i+K}\quad(K=10)$$
> 朝全局均值 (\bar g) 收缩，修正「购买人数=1 时复购率虚高为 1.0」导致单买家商品霸榜的问题。修正后 CRITIC 权重均衡，热度 Top10 为新鲜蛋品、香蕉、水果等真实高频生鲜。

## 6.3 商品价格带

在每个小类内部计算商品价格带：

| 价格分位 | 价格带 |
| -------- | ------ |
| 0%—33%   | 低价带 |
| 33%—66%  | 中价带 |
| 66%—100% | 高价带 |

这样可以用于：

- 价格敏感用户推荐低价带商品；
- 高价值用户推荐高价带或升级商品；
- 营销者分析不同价格带商品的销售贡献。

# 7. 顾客分群模型：UMAP + HDBSCAN / GMM 软分群

## 7.1 为什么不用 K-Means

K-Means 对球形簇假设较强，需要预先指定聚类数，并且对异常值敏感。超市顾客行为往往存在长尾分布，例如少数高消费用户、强促销用户、生鲜高频用户、低频偶发用户等，群体形状不一定规则。因此本项目不以 K-Means 作为主分群模型。

## 7.2 自然结构识别：UMAP + HDBSCAN

采用：

> **UMAP 降维 + HDBSCAN 密度聚类**（用于识别数据的自然宏观结构、并标记边界/噪声用户）

### UMAP 作用

UMAP 将高维顾客画像压缩到低维空间，保留顾客之间的局部邻近关系。

输入特征包括：

```text
R_最近购买天数
F_购买频次
M_消费金额
客单价
类目熵
大类消费占比
中类消费占比
小类购买数量
促销金额占比
促销购买次数占比
低价带消费占比
高价带消费占比
生鲜消费占比
复购紧迫度均值
```

### HDBSCAN 作用

HDBSCAN 可以自动识别密度不同的顾客群体，并将难以归类的用户标记为噪声点或边界用户。它适合处理顾客行为不均衡、长尾明显的零售数据。

相比传统聚类，它的优势是：

1. 不需要手动指定聚类数；
2. 能识别非球形顾客群体；
3. 能处理异常顾客；
4. 能输出聚类稳定性；
5. 更适合高价值长尾用户识别。

## 7.3 落地主模型：AutoEncoder + GMM 软分群

HDBSCAN 适合发现自然群体，但营销场景中，一个用户可能同时具备多个属性。例如：

- 既是生鲜高频用户；
- 又是促销敏感用户；
- 还可能是高价值用户。

因此可引入 **GMM 高斯混合模型** 做软分群，输出用户属于各群体的概率：
$$
P(z=k|x_u)
$$
最终可以形成：

| 顾客   | 高价值稳定型 | 生鲜高频型 | 促销敏感型 | 流失预警型 |
| ------ | ------------ | ---------- | ---------- | ---------- |
| 用户 A | 0.62         | 0.24       | 0.10       | 0.04       |
| 用户 B | 0.11         | 0.18       | 0.58       | 0.13       |

这比硬分类更适合营销者做精细化运营。

> **实验落地**：直接在原始画像空间用 GMM 软分群轮廓系数偏低（0.021）。改用 **AutoEncoder（16→32→16→5 瓶颈）先做非线性去噪压缩，再在 5 维嵌入上用 GMM(k=7，BIC 选定)** 软分群，轮廓系数提升至 0.106、CH 指数 182→263，且输出每个顾客对 7 个群体的软归属概率 (P(z=k\mid x_u))。这是最终落地的精细化营销分群主模型。

## 7.4 推荐分群标签

最终建议形成以下顾客群体：

| 群体名称     | 主要特征                         | 消费者侧推荐             | 营销者侧策略                 |
| ------------ | -------------------------------- | ------------------------ | ---------------------------- |
| 高价值稳定型 | 消费金额高、购买频次高、近期活跃 | 高品质商品、新品、组合包 | 会员运营、积分加倍、专属权益 |
| 生鲜高频型   | 蔬果、日配、生鲜占比高           | 生鲜补货提醒、每日鲜品   | 高频触达、周期性生鲜券       |
| 促销敏感型   | 促销占比高、低价带占比高         | 促销商品、满减商品       | 优惠券、限时折扣、组合促销   |
| 类目集中型   | 类目熵低、偏好稳定               | 同类替代、同类升级       | 精准类目营销                 |
| 跨类探索型   | 类目熵高、购买范围广             | 跨品类组合推荐           | 新品试用、联动促销           |
| 流失预警型   | 最近购买间隔长、历史消费不低     | 常购商品提醒、召回优惠   | 召回券、短信触达             |
| 低频偶发型   | 购买少、金额低、偏好弱           | 热门刚需商品             | 低门槛优惠、基础促活         |

## 7.5 实验结论与最终方案

四个分群实验（统一在原始标准化特征空间公平评估）结果：

| 实验 | 模型 | 轮廓系数↑ | DB指数↓ | CH指数↑ | 群体数 | 噪声 |
| ---- | ---- | -------- | ------- | ------- | ----- | ---- |
| E1 | GMM | 0.021 | 3.37 | 182 | 7 | 0 |
| E2 | UMAP+HDBSCAN | 0.137 | 1.82 | 298 | 2 | 1.4% |
| E3 | **AutoEncoder+GMM** | 0.106 | 2.21 | 263 | 7 | 0 |
| E4 | UMAP+HDBSCAN+GMM 融合 | 0.132 | 1.84 | 291 | 2 | 0 |

**关键发现**：HDBSCAN 在该数据上要么产出 23 个碎簇（35% 噪声）、要么 2 个干净大群——不存在自然的 5–7 群结构。结合 E1 裸 GMM 轮廓仅 0.021，说明**零售顾客行为本质是双模态连续谱**，而非离散簇，这是真实的数据结论而非模型失败。

> **最终方案**：以 **AE+GMM（7 营销群体）作为可落地软分群主模型**（k=7 时指标全面优于裸 GMM）；以 **UMAP+HDBSCAN（2 个稳健宏观群 + 边界用户）验证自然结构、识别需单独触达的边界用户**。Top2 高价值群体（占 46% 顾客）贡献 75% 销售额，帕累托效应显著。

# 8. 商品关联模型：FP-Growth + 高效用项集挖掘

## 8.1 模型目标

商品关联模型服务两类用户：

| 服务对象   | 目标                                   |
| ---------- | -------------------------------------- |
| 消费者用户 | 买了 A 后推荐可能需要的 B              |
| 营销者用户 | 找出适合组合促销、联动陈列的商品或类目 |

## 8.2 FP-Growth 关联规则

相比 Apriori，FP-Growth 不需要反复扫描候选项集，适合更高效地挖掘购物篮规则。

对于规则：
$$
A\Rightarrow B
$$
支持度：
$$
support(A\Rightarrow B)=P(A\cup B)
$$
置信度：
$$
confidence(A\Rightarrow B)=P(B|A)
$$
提升度：
$$
lift(A\Rightarrow B)=\frac{P(A\cup B)}{P(A)P(B)}
$$
筛选规则：

| 指标           | 商品级建议 | 小类级建议 |
| -------------- | ---------- | ---------- |
| min_support    | 0.003—0.01 | 0.01—0.03  |
| min_confidence | ≥ 0.20     | ≥ 0.25     |
| min_lift       | ≥ 1.10     | ≥ 1.15     |
| 后项数量       | 1          | 1          |

## 8.3 高效用项集挖掘 HUIM

传统关联规则只看“是否共现”，不考虑销售金额。本项目数据有销售金额和销售数量，因此可以引入：

> **High Utility Itemset Mining，高效用项集挖掘**

定义商品 (i) 在购物篮 (b) 中的效用：
$$
u(i,b)=quantity_{i,b}\cdot unit_price_{i,b}
$$
项集 (X) 的总效用：
$$
U(X)=\sum_{b\in \mathcal{B}}\sum_{i\in X}u(i,b)I(X\subseteq b)
$$
高效用项集不仅要求出现频繁，还要求销售贡献高。

这样可以帮助营销者识别：

- 高频但低金额组合；
- 低频但高价值组合；
- 真正值得做组合促销的商品包。

## 8.4 层级关联规则

利用三级类目结构，建立三层规则：

```text
中类规则：适合经营分析
小类规则：适合促销组合
商品规则：适合消费者推荐
```

例如：

| 层级   | 示例            | 用途           |
| ------ | --------------- | -------------- |
| 中类级 | 蔬菜 → 调味品   | 陈列优化       |
| 小类级 | 花果 → 酱菜     | 组合促销       |
| 商品级 | 商品 A → 商品 B | 个性化加购推荐 |

最终采用：

> 商品级规则优先，小类级规则兜底，中类级规则解释经营逻辑。

# 9. 消费者用户侧推荐模型

消费者侧目标是为每个顾客生成个性化推荐清单，并给出推荐理由。

## 9.1 消费者侧推荐任务

对顾客 (u)，输出：
$$
Rec_u=\{i_1,i_2,\cdots,i_K\}
$$
每条推荐包含：

| 字段                     | 含义         |
| ------------------------ | ------------ |
| user_id                  | 顾客编号     |
| item_id                  | 推荐商品     |
| cat_l1 / cat_l2 / cat_l3 | 商品所属类目 |
| score                    | 推荐得分     |
| reason                   | 推荐理由     |
| strategy                 | 推荐策略类型 |

## 9.2 消费者侧候选召回

### 9.2.1 图推荐召回：二部图嵌入（LightGCN 同源）

构建用户—商品二部图：
$$
G=(U,I,E)
$$
其中：

- (U)：顾客集合；
- (I)：商品集合；
- (E)：用户购买商品形成的边。

边权可以定义为：
$$
w_{u,i}=
\log(1+freq_{u,i})
+\log(1+amount_{u,i})
+e^{-\lambda \Delta t_{u,i}}
$$
图推荐模型通过用户—商品交互图传播偏好，适合发现用户没有买过但可能感兴趣的商品。

消费者侧推荐得分：
$$
Score_{graph}(u,i)=e_u^T e_i
$$
其中 (e_u) 是用户嵌入，(e_i) 是商品嵌入。

> **实验落地**：边权矩阵 (M=[w_{u,i}]) 经**截断 SVD** 分解 (M\approx U\Sigma V^T)，取 (e_u=U\sqrt\Sigma)、(e_i=V\sqrt\Sigma)（48 维）。这是用户-商品二部图邻接矩阵的低秩图嵌入，与 LightGCN「无特征变换、纯邻域传播」思想同源，但无需迭代训练，轻量稳健。实验中该图嵌入召回是**最强单路召回**（HitRate@10=0.443，NDCG@10=0.113，全面第一）。后续可升级为可训练的 LightGCN（BPR 损失 + 多层图卷积）进一步提升个性化。

### 9.2.2 关联规则召回

如果顾客最近购买过商品或小类 (A)，并存在规则：
$$
A\Rightarrow B
$$
则推荐 (B)。

规则得分可以定义为：
$$
Score_{rule}(u,i)=
confidence(A\Rightarrow i)\cdot lift(A\Rightarrow i)
$$

### 9.2.3 类目偏好召回

根据顾客历史偏好类目推荐同类商品：
$$
Score_{cat}(u,i)=p_{u,c(i)}\cdot Popularity_i
$$
其中：

- (p_{u,c(i)})：顾客对商品所属类目的偏好；
- (Popularity_i)：商品综合热度。

### 9.2.4 复购周期召回

对生鲜、粮油、日配等复购商品，计算复购紧迫度：
$$
Score_{cycle}(u,i)=Need_{u,c(i)}
$$
若顾客快到该小类复购时间，则推荐该小类下常买商品或热门商品。

### 9.2.5 促销适配召回

若顾客促销敏感，则提升促销商品推荐：
$$
Score_{promo}(u,i)=PromoSensitivity_u\cdot I \quad (i为促销)
$$

### 9.2.6 价格带匹配召回

顾客价格偏好与商品价格带匹配度：
$$
Score_{price}(u,i)=
1-|PricePref_u-PriceRank_i|
$$
其中：

- (PricePref_u)：顾客偏好价格分位；
- (PriceRank_i)：商品在所属小类中的价格分位。

# 10. 消费者侧推荐融合：CRITIC-TOPSIS 自适应排序

## 10.1 不人工初设权重

本项目采用：

> **CRITIC 权重法 + TOPSIS 综合排序**

原因：

1. 不依赖主观经验；
2. 能根据数据自动确定指标重要性；
3. 能处理多个推荐指标；
4. 适合做可解释推荐排序；
5. 能避免某个指标与其他指标高度重复。

## 10.2 推荐指标矩阵

对每个候选商品 (i)，构建指标向量：
$$
X_i=
[
Score_{graph},
Score_{rule},
Score_{cat},
Score_{cycle},
Score_{promo},
Score_{price},
Popularity,
Diversity
]
$$
形成候选商品评价矩阵：
$$
X=
[x_{ij}]_{m\times n}
$$
其中：

- (m)：候选商品数量；
- (n)：评价指标数量。

## 10.3 指标标准化

对于正向指标：
$$
z_{ij}=
\frac{x_{ij}-\min(x_j)}
{\max(x_j)-\min(x_j)}
$$
对于负向指标，例如重复推荐惩罚：
$$
z_{ij}=
\frac{\max(x_j)-x_{ij}}
{\max(x_j)-\min(x_j)}
$$

## 10.4 CRITIC 计算权重

CRITIC 方法同时考虑指标波动性和指标相关性。

指标 (j) 的标准差：
$$
\sigma_j
$$
指标 (j) 与指标 (k) 的 Pearson 相关系数：
$$
r_{jk}
$$
指标信息量：
$$
C_j=
\sigma_j\sum_{k=1}^{n}(1-r_{jk})
$$
权重：
$$
w_j=
\frac{C_j}{\sum_{j=1}^{n}C_j}
$$
这样，如果某个指标波动大、且与其他指标不重复，它的权重会更高；如果某个指标与其他指标高度相关，则权重会被自动降低。

> **实验修正（可靠度校准）**：纯无监督 CRITIC 会给「高方差但与购买无关」的信号（如价格匹配）过高权重，且 TOPSIS「逼近正理想解」会埋没仅在强信号（图嵌入）上突出的优质候选，导致融合一度劣于单路召回。故引入**训练集内部时间切分**（1–2 月训练 / 3 月验证）评估每个召回信号的预测可靠度 (rel_j)，最终权重取
> $$w_j=\frac{C_j\cdot rel_j}{\sum_k C_k\cdot rel_k}$$
> 即 **CRITIC 信息量 × 预测可靠度**。无关信号被自动压低（价格 0.26、促销 0.14），强预测信号被提升（图嵌入 / 热度 1.0）。这仍是数据驱动、不依赖人工设定权重。修正后融合精度追平最强单路召回，且覆盖率、多样性、可解释性全面领先。

## 10.5 TOPSIS 计算最终推荐得分

加权标准化矩阵：
$$
v_{ij}=w_jz_{ij}
$$
正理想解：
$$
A^+=\{\max(v_{ij})\}
$$
负理想解：
$$
A^-=\{\min(v_{ij})\}
$$
候选商品到正理想解距离：
$$
D_i^+=
\sqrt{
\sum_j(v_{ij}-A_j^+)^2
}
$$
到负理想解距离：
$$
D_i^-=
\sqrt{
\sum_j(v_{ij}-A_j^-)^2
}
$$
TOPSIS 贴近度：
$$
C_i=
\frac{D_i^-}{D_i^+ + D_i^-}
$$
最终按照 (C_i) 从高到低排序，输出 Top-K 推荐。

# 11. 营销者用户侧决策模型

营销者侧不是给单个消费者推荐商品，而是回答：

1. 哪类顾客值得重点运营？
2. 哪些商品适合组合促销？
3. 哪些类目有增长空间？
4. 哪些促销策略真正有效？
5. 应该给什么用户发什么券？

## 11.1 营销者侧顾客价值识别

使用 HDBSCAN / GMM 得到顾客群体后，进一步用 TOPSIS 评估每个群体的营销价值。

群体评价指标包括：

| 指标             | 方向       |
| ---------------- | ---------- |
| 群体总销售额     | 正向       |
| 群体人均消费金额 | 正向       |
| 群体购买频次     | 正向       |
| 群体复购率       | 正向       |
| 群体促销响应率   | 正向       |
| 群体流失风险     | 负向       |
| 群体价格敏感度   | 视策略而定 |

对每个群体 (g) 形成评价矩阵：
$$
X_g=
[
Sales_g,
AvgM_g,
Freq_g,
Repurchase_g,
PromoResponse_g,
ChurnRisk_g
]
$$
再用 CRITIC-TOPSIS 排序，得到：
$$
ValueRank_g
$$
营销者可以据此识别：

- 高价值保留群体；
- 高促销响应群体；
- 高流失风险群体；
- 高潜力增长群体。

## 11.2 商品组合促销决策

结合 FP-Growth 和 HUIM，构建组合促销候选：
$$
Bundle=\{i_1,i_2,\cdots,i_k\}
$$
对每个组合计算：

| 指标          | 含义           |
| ------------- | -------------- |
| support       | 组合出现频率   |
| confidence    | 组合转化概率   |
| lift          | 组合关联强度   |
| utility       | 组合销售贡献   |
| promo_lift    | 促销提升效果   |
| category_span | 是否跨类目     |
| audience_size | 可触达顾客规模 |

再用 TOPSIS 计算组合价值：
$$
BundleScore_b=TOPSIS(support,confidence,lift,utility,promo_{lift},audience_{size})
$$
最终输出：

| 组合            | 推荐原因       | 适合人群   | 策略         |
| --------------- | -------------- | ---------- | ------------ |
| 蔬果 + 调味品   | 共购强、金额高 | 生鲜高频型 | 生鲜满减组合 |
| 粮油 + 日配     | 刚需复购       | 家庭型用户 | 周末囤货券   |
| 休闲食品 + 饮品 | 跨品类共购     | 跨类探索型 | 第二件折扣   |

## 11.3 促销响应分析

促销效果不能只看促销商品卖得多不多，还要分析不同群体是否因为促销发生购买提升。

定义群体促销销售占比：
$$
PromoRate_g=
\frac{
\sum_{u\in g} amount_{u,promo}
}{
\sum_{u\in g} amount_u
}
$$
定义促销提升度：
$$
PromoLift_g=
\frac{
AvgSales_{g,promo}
}{
AvgSales_{g,nonpromo}
}
$$
如果：
$$
PromoLift_g>1
$$
说明该群体对促销有明显响应。

### 11.3.1 促销因果效应：双重机器学习（DML）

朴素 PromoLift 存在严重**混淆偏差**：被促销的商品本身在品类、价格、顾客构成上系统性不同，直接比较「促销 vs 非促销」的销售均值会把「商品差异」误当成「促销效果」。

为得到去偏的因果结论，引入 **双重机器学习（Double Machine Learning, Chernozhukov 2018）**。设部分线性模型：
$$
Y=\theta\cdot T+g(X)+\varepsilon,\qquad T=m(X)+\nu
$$
其中 (T) = 是否促销，(Y) = 销售金额，(X) = 混淆变量（大类、单价、小类内价格分位、时间、顾客 RFM 等）。采用 **K 折交叉拟合** 估计 nuisance：(\hat g(X)=E[Y\mid X])、(\hat m(X)=E[T\mid X])（用 LightGBM，out-of-fold 预测消除过拟合偏差），再做残差正交化（Robinson）：
$$
\tilde Y=Y-\hat g(X),\quad \tilde T=T-\hat m(X),\qquad
\hat\theta=\frac{\sum \tilde T\tilde Y}{\sum \tilde T^2}
$$
并由影响函数给出 (\hat\theta) 的渐近方差与 95% 置信区间。按群体分组得到**异质处理效应 CATE** (\hat\theta_g)。

> **实验结论（符号反转）**：朴素均值差显示促销使笔均金额 **+7.18 元**，但 DML 去偏后 **ATE = −2.50 元/笔**（95%CI [−4.95, −0.04]，显著）——促销实际**侵蚀**笔均收入（折扣效应盖过走量）。CATE 显示仅「促销敏感型2」群体因果响应为正（+1.17 元），对「高价值稳定型」促销显著毁利（−1.45 元）。**营销含义**：应停止全员撒券，转向精准投放促销敏感群体、对高价值群体用权益替代降价。

营销策略：

| 群体         | 促销响应 | 策略                       |
| ------------ | -------- | -------------------------- |
| 促销敏感型   | 高       | 发券、满减、限时折扣       |
| 高价值稳定型 | 中       | 权益型促销，不建议过度降价 |
| 生鲜高频型   | 高       | 生鲜周期券、早晚市优惠     |
| 流失预警型   | 不确定   | 召回券 + 常购商品提醒      |
| 低频偶发型   | 低       | 低门槛新人券或热门刚需推荐 |

## 11.4 品类经营分析

对大类、中类、小类分别计算：
$$
Contribution_c=
\frac{Sales_c}{TotalSales}
$$
增长率：
$$
Growth_c=
\frac{Sales_{c,t}-Sales_{c,t-1}}
{Sales_{c,t-1}}
$$
]

促销依赖度：
$$
PromoDepend_c=
\frac{Sales_{c,promo}}{Sales_c}
$$
复购贡献：
$$
Repurchase_c=
\frac{重复购买该类目的顾客数}{购买该类目的顾客数}
$$
根据这些指标，将类目划分为：

| 类目类型       | 特征             | 策略               |
| -------------- | ---------------- | ------------------ |
| 核心稳定类目   | 高销售、高复购   | 保持供应、会员权益 |
| 增长潜力类目   | 增长快、覆盖提升 | 增加曝光、交叉推荐 |
| 促销依赖类目   | 促销占比高       | 控制促销成本       |
| 高价值低频类目 | 金额高、频次低   | 精准推荐           |
| 长尾弱势类目   | 销售低、复购低   | 清理、打包、降库存 |

# 13. 模型评价指标

## 13.1 消费者推荐评价指标

采用时间切分验证。用前一段时间训练模型，用后一段时间真实购买记录验证推荐是否命中。

### Precision@K

$$
Precision@K=
\frac{
TopK中被真实购买的商品数
}{
K
}
$$

### Recall@K

$$
Recall@K=
\frac{
TopK中被真实购买的商品数
}{
用户真实购买商品数
}
$$

### HitRate@K

$$
HitRate@K=
\begin{cases}
1, & TopK中至少命中一个商品\\
0, & 否则
\end{cases}
$$

### NDCG@K

用于评价推荐排序是否合理：
$$
NDCG@K=
\frac{DCG@K}{IDCG@K}
$$
同时计算：

| 指标              | 作用                           |
| ----------------- | ------------------------------ |
| Coverage@K        | 推荐覆盖率，避免只推荐热门商品 |
| Diversity@K       | 推荐多样性                     |
| Novelty@K         | 新颖性                         |
| PromoMatchRate    | 促销匹配率                     |
| RepurchaseHitRate | 复购命中率                     |
| CategoryHitRate   | 类目命中率                     |

## 13.2 营销者策略评价指标

| 指标             | 含义                         |
| ---------------- | ---------------------------- |
| 群体区分度       | 不同顾客群体画像是否明显     |
| 群体稳定性       | 聚类结果是否稳定             |
| 策略覆盖人数     | 某策略可覆盖多少顾客         |
| 组合促销价值     | 商品组合的销售贡献与关联强度 |
| 促销提升度       | 促销是否提高销售表现         |
| 高价值顾客识别率 | 是否识别出主要贡献顾客       |
| 类目经营解释性   | 品类策略是否清晰可解释       |

# 17. 最终模型方案总结

本项目最终采用：

> **AutoEncoder-GMM 软分群（UMAP-HDBSCAN 验证自然结构）+ FP-Growth 层级关联规则 + 高效用项集挖掘 + 二部图嵌入召回 + CRITIC-TOPSIS（可靠度校准）自适应排序 + DML 促销因果推断**

整体模型分为两条主线。

## 17.1 消费者用户侧

目标：

> 给每个顾客推荐更适合他的商品。

流程：

```text
顾客历史购买记录
    ↓
顾客画像：
RFM + 类目偏好 + 促销敏感度 + 价格敏感度 + 复购周期
    ↓
候选召回：
图推荐 + 关联规则 + 类目偏好 + 复购周期 + 促销适配
    ↓
CRITIC 自动计算指标权重
    ↓
TOPSIS 综合排序
    ↓
输出 Top-K 商品推荐 + 推荐理由
```

消费者侧输出的是：

- 个性化商品推荐；
- 促销商品推荐；
- 复购提醒；
- 推荐理由解释。

## 17.2 营销者用户侧

目标：

> 帮助营销者决定推什么、推给谁、怎么推。

流程：

```text
顾客画像 + 商品画像 + 销售行为
    ↓
UMAP + HDBSCAN 识别自然宏观结构（验证 + 边界用户）
    ↓
AutoEncoder + GMM 输出 7 营销群体软分群概率
    ↓
FP-Growth 挖掘共购规则
    ↓
高效用项集挖掘识别高价值商品组合
    ↓
CRITIC-TOPSIS 评估群体价值、商品组合价值、促销策略价值
    ↓
输出营销策略报告
```

营销者侧输出的是：

- 顾客群体划分；
- 高价值顾客群体；
- 流失预警顾客群体；
- 商品组合促销策略；
- 促销敏感人群识别；
- 类目经营建议；
- 不同群体的差异化营销方案。

# 18. 本方案的创新点

## 创新点一：不用传统 K-Means，采用密度聚类 + 深度嵌入软分群

不以 K-Means 作为主模型。用 UMAP+HDBSCAN 识别数据的自然密度结构（实验证实为双模态连续谱），再用 **AutoEncoder 非线性降噪嵌入 + GMM** 给出可落地的 7 营销群体软分群，适合长尾明显、群体形状复杂的超市消费数据。

## 创新点二：使用 GMM 软分群补充硬标签

一个顾客可以同时具有多个群体属性，输出对各群体的归属概率 (P(z=k\mid x_u))，避免简单粗暴地把用户只归为一类。

## 创新点三：消费者侧与营销者侧双目标建模

消费者侧关注“推荐什么”，营销者侧关注“如何运营”，避免推荐系统只服务单一目标。

## 创新点四：引入高效用项集挖掘

不仅看商品是否共购，还看商品组合是否真正贡献销售金额。

## 创新点五：推荐权重不人工设置

使用 CRITIC 根据指标波动性和 Pearson 相关性自动计算权重，再用 TOPSIS 排序，避免主观设定权重。

## 创新点六：融合复购周期、促销敏感度和价格带偏好

更符合超市实际消费场景，尤其适合生鲜、粮油、日配等高复购品类。

## 创新点七：推荐结果可解释

每个推荐结果都能说明来源，例如：

- 因为你常买该类目；
- 因为你快到复购周期；
- 因为相似顾客也买了；
- 因为该商品与近期购买商品强关联；
- 因为该商品符合你的促销偏好和价格带。

## 创新点八：促销响应引入因果推断（DML）

不止步于相关性分析。用双重机器学习在控制混淆变量后估计促销的**去偏因果效应（ATE/CATE）**，纠正朴素 PromoLift 的混淆偏差（实验中促销效应符号由 +7.18 被纠正为 −2.50 元/笔），为「该不该促、对谁促」提供因果级证据，支撑精准发券而非全员撒券。

# 19. 项目拓展与深化模块（v2 新增）

在双侧主模型之外，进一步在推荐、营销、运营三条线做深化与广度拓展，全部配有独立实验报告（`experimental_docs/report_07~12`）。

## 19.1 LightGCN 图神经推荐（深化召回）

将图召回从截断 SVD 升级为可训练的 **LightGCN**（BPR 损失 + 多层邻域传播）。严格"发现型"协议（全目录排序、剔除已购）下，LightGCN 相对 SVD 嵌入 HitRate@10 +70%、NDCG@10 +140%，验证非线性高阶图传播的价值。脚本 `exp_lightgcn.py`。

## 19.2 Uplift 建模与个性化发券名单（深化营销）

将 §11.3 的群体级 CATE 下沉到**个体处理效应(ITE)**。对比 T-learner / R-learner / **DR-learner（双重稳健 AIPW）**，最终采用 DR-learner，AUUC 达随机发券基线的 **1.70 倍**。输出可执行发券名单：uplift>0 发券、uplift≤0 改用权益。脚本 `exp_uplift.py`。

## 19.3 顾客流失预测 + SHAP（广度）

特征窗 1–2 月、标签窗 3–4 月，LightGBM 预测流失，**ROC-AUC 0.78**。SHAP 解释显示购买频次、活跃跨度、最近购买间隔为主因。输出高价值高风险**优先召回名单**，把"流失预警型"从事后画像升级为事前概率预警。脚本 `exp_churn.py`。

## 19.4 销售时间序列预测（广度）

日销售额**季节分解**（趋势含春节高峰、规整周季节）+ **ACF/PACF**（确认周期 7 天）+ **Holt-Winters** 预测，RMSE 优于季节朴素基线 13.7%。支撑备货与排班。脚本 `exp_timeseries.py`。

## 19.5 顾客生命周期价值 CLV（广度）

**BG/NBD + Gamma-Gamma** 概率模型预测未来 90 天购买次数、存活概率与价值。校准/留出验证预测-实际相关 0.68、总量误差 3%。Top10% 顾客贡献 56.9% CLV。把静态 RFM 升级为前瞻性价值预测。脚本 `exp_clv.py`。

## 19.6 价格弹性与鲁棒性检验（深化）

- **价格弹性**：DML 估计 (\ln Q) 对 (\ln P) 的去偏弹性，整体 −0.25、各品类普遍缺乏弹性（日配/休闲最敏感）→ 指导差异化定价。
- **规则显著性**：43 条小类规则经卡方 + Bonferroni 校正，36 条显著，证明非偶然。
- **聚类稳定性**：bootstrap ARI 0.48±0.12，印证连续谱特性（细分群中等稳定、宏观群稳定）。

脚本 `exp_elasticity_robust.py`。

## 19.7 蒙特卡洛合成数据验证（方法学背书）

真实数据无标准答案，难以判断估计是否正确。构造**已知真值**的生成过程（5 类顾客原型 + 注入促销因果效应 β=0.40、价格弹性 ε=−0.60，且促销分配受类目/价格混淆），让本项目方法盲跑还原：
- **DML 近乎完美还原真值**：促销效应 0.396（真值 0.40）、弹性 −0.599（真值 −0.60）；而朴素估计被混淆压低到 0.18 → 直接验证 §11.3 与弹性结论的可信度；
- **分群在信号充足时有效**：高活跃顾客 ARI 0.63（全体 0.44，差距源于低频顾客稀疏信号），解释真实数据 ARI 0.48 的来源。

脚本 `exp_montecarlo.py`，15 次重复，估计标准差 <0.01，结论可复现。

## 19.8 创新点（拓展）

- **创新点九**：推荐召回端落地可训练 LightGCN，并与线性图嵌入做同协议对比量化增益。
- **创新点十**：从群体 CATE 到个体 uplift（DR-learner）闭环，产出量化增益的发券名单。
- **创新点十一**：因果方法二次落地——用 DML 估计需求价格弹性，与促销因果效应共同支撑「定价 + 促销」一体化决策。
- **创新点十二**：对关键结论做统计显著性与稳定性检验（卡方/Bonferroni/bootstrap-ARI），强化结论可信度。
- **创新点十三**：蒙特卡洛合成已知真值数据，对全流程做白盒可还原性验证，为方法学正确性提供背书。

------

这版模型的核心思路可以概括为：

> **对消费者，做个性化推荐；对营销者，做策略化推荐。
> 对推荐排序，不人工拍权重，而用 CRITIC-TOPSIS（可靠度校准）数据驱动计算。
> 对顾客分群，不使用传统 K-Means，而用 AutoEncoder-GMM 软分群与 UMAP-HDBSCAN 结构验证，形成更适合零售长尾数据的分群体系。
> 对促销决策，不止于相关性，而用双重机器学习给出去偏的因果效应，支撑精准营销。**
