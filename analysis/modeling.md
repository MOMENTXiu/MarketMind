# 超市 AI 营销系统——项目思路说明（Markdown 草案）

> 角色：超市 IT 经理 / 资深算法分析师
>  数据：`dataset.csv`，约 1 万行订单明细记录（含订单 ID、订单日期、客户 ID、产品 ID、销售额、折扣、利润等）

------

## 1. 项目目标与分解

- **总体目标**：基于历史订单数据，构建一套“AI 营销系统”，实现：
  1. 通过**关联规则挖掘**制定商品组合促销策略；
  2. 通过**预测模型**预测未来销售额和利润；
  3. 通过**客户聚类**进行客户分群，制定差异化营销策略；
  4. 将上述分析结果通过**语音合成**自动播报。

------

## 2. 数据概览与预处理思路

### 2.1 样本字段（依据截图推断）

- 行 ID（行号）
- 订单 ID（如：`US-202023-1357144`）
- 订单日期、发货日期
- 邮寄方式（标准级 / 二级…）
- 客户 ID、客户名称、客户细分（消费者 / 公司 / 小型企业…）
- 城市、省份/自治区、国家、地区
- 产品 ID、类别、子类别、产品名称
- 销售额、数量、折扣、利润

### 2.2 预处理要点

1. **类型转换**
   - 日期字段转为 `datetime`，构造 `年/月/周/季度`、`是否节假日` 等时间特征；
   - 销售额、数量、折扣、利润转为数值型。
2. **数据清洗**
   - 缺失值检测与填补（如客户细分缺失、产品名称缺失等）；
   - 异常值检测（极端折扣、负利润极值等）；
   - 去重：按「订单 ID + 产品 ID」去重，防止重复记录。
3. **核心视图构建**
   - **交易视图（订单级）**：
      按 `订单 ID` 分组，将同一订单的多个商品行合并为一条“购物篮记录”
      （后续用于关联规则挖掘）。
   - **时间序列视图**：
      按日/周/月聚合销售额与利润，形成 `date -> sales, profit` 的时间序列表。
   - **客户视图**：
      按 `客户 ID` 聚合，构建每个客户的 RFM 与品类消费特征，用于聚类。

## 4. 模块一：商品关联规则与促销策略

### 4.1 问题拆解

> 需求：合并同一订单 ID 的记录，形成购物清单集合，挖掘“商品作为后项的关联规则”，指导组合促销。

典型问题形式：

- **如果顾客购买了 A、B，那么他大概率会再买 C**
- 对于指定商品（如高利润品或滞销品）作为**后项**：
  - 找到“哪些前置商品组合能有效带动该商品”。

### 4.2 数据建模流程

1. **构建购物篮数据**
   - 对每个 `订单 ID`：
     - 收集该订单中所有 `产品 ID` 或 `产品名称`；
     - 形成交易记录：`Transaction_i = { item1, item2, ... }`。
2. **编码与稀疏表示**
   - 将产品映射为 one-hot / 哑变量；或使用事务列表形式供算法读取；
   - 为提高效率，过滤低频商品（如支持度 < 0.5%）。
3. **关联规则算法**
   - 候选算法：`Apriori`、`FP-Growth`；
   - 指标：
     - 支持度 Support
     - 置信度 Confidence
     - 提升度 Lift（>1 代表正相关）
   - 针对“后项指定商品”的规则过滤：
     - 规则形如：`前项 X -> 后项 Y`，其中 `Y` 属于指定商品集合（如利润高的品）。
4. **策略设计**
   - 从规则中提取：
     - 高频组合商品；
     - 可搭配捆绑销售的高利润商品；
     - 交叉销售机会（不同类别之间的关联）。
   - 形成策略示例：
     - **满额赠品**：购买 A+B 时推荐 C；
     - **组合打折包**：将 {A, B, C} 组合成套餐；
     - **推荐系统触发逻辑**：顾客加入 A 到购物车时，实时推荐 B,C。

## 5. 模块二：销售额与利润预测

### 5.1 问题拆解

> 需求：预测未来一段时间（如未来 N 周 / N 月）的总销售额与总利润。

可细化为三层：

1. **总体预测**：全店日/周/月销售额、利润；
2. **维度预测**：按品类 / 子类别 / 区域的销售与利润；
3. **场景应用**：备货计划、预算制定、营销投入 ROI 评估。

### 5.2 时间序列构建

1. 聚合规则：
   - 按 `订单日期`，按日/周/月分别聚合：
     - `sales_t = Σ 销售额`
     - `profit_t = Σ 利润`
   - 可对不同维度分别聚合（如按品类、地区拆分出多个时间序列）。
2. 特征工程：
   - **时间特征**：星期几、月份、是否周末、是否节假日；
   - **滞后特征**：`sales_{t-1}, sales_{t-7}, sales_{t-30}` 等；
   - **滑动统计**：7 日 / 30 日滑动平均、滑动标准差；
   - **事件特征**：促销活动、重大节日标记。

### 5.3 模型选择

- 传统时间序列：
  - **ARIMA / SARIMA**：适合稳定单变量序列；
  - **Prophet**：处理季节性、节假日效果较好。
- 机器学习 / 深度学习回归：
  - **XGBoost / LightGBM**：处理多特征回归，适合有大量外部特征场景；
  - **LSTM / GRU**：适合捕捉时间依赖性、非线性关系。
- 多步预测策略：
  - 递推预测（一步步往前滚）；
  - 直接预测（模型一次输出未来多步）。

### 5.4 评估指标与部署

- 指标：RMSE，MAE，MAPE，R²
- 交叉验证：滚动时间窗口切分训练集/验证集/测试集
- 上线后：
  - 每天定时更新最近数据，重新训练/增量训练；
  - 自动生成“下周 / 下月销售额&利润预测”摘要文本。

## 6. 模块三：客户聚类与精细化营销

### 6.1 问题拆解

> 需求：对客户进行聚类分析，形成客户分群，制定针对性营销策略。

思路：以客户 ID 为粒度，通过 RFM 模型 + 消费偏好特征，进行聚类、命名分群、制定策略。

### 6.2 特征设计

1. **RFM 特征**
   - R（Recency）：最近一次消费距今的天数；
   - F（Frequency）：一定时间窗口内消费次数；
   - M（Monetary）：一定时间窗口内消费总金额 / 总利润。
2. **品类偏好与行为特征**
   - 各类别/子类别的购买次数或金额占比；
   - 平均折扣、对折扣的敏感度（折扣高时是否更频繁购买）；
   - 订单平均客单价、平均购买件数。
3. **数据标准化**
   - 由于 R/F/M 数量级不同，使用 Z-score 或 Min-Max 进行标准化。

### 6.3 聚类算法

- **K-Means**：简单高效，适用于中等维度、数值特征；
- **层次聚类**：可辅助判断合适簇数；
- **GMM（高斯混合）**：可以得到软聚类概率，适合边界模糊的客户。

### 6.4 分群解读与策略输出

样例分群（示意）：

1. **高价值忠诚客户**：高 F，高 M，低 R
   - 策略：积分体系、会员专属优惠、VIP 客服。
2. **潜力成长客户**：中 F，中 M，R 中等
   - 策略：有门槛优惠券、组合推荐。
3. **价格敏感型客户**：高折扣敏感、客单价偏低
   - 策略：折扣券、满减活动、折扣专区推送。
4. **流失预警客户**：R 高、F 低
   - 策略：召回短信/邮件、限时优惠。

## 7. 模块四：语音合成播报

### 7.1 需求说明

> 需求：将 1~3 模块的结果通过语音合成算法播报出来。

播报内容示例：

- 关联规则&促销：

  > “根据最近三个月的购物篮分析，购买打印机的顾客中，有 45% 会同时购买 A4 纸，我们建议将两者设置为关联促销组合。”

- 销售预测：

  > “模型预测，下周门店总销售额为 120 万元，利润约 15 万元，较上周预计增长 8%。”

- 客户分群：

  > “当前共有 520 名高价值客户，预计贡献总销售额的 42%，建议重点投放会员专属活动。”

### 7.2 文本生成与模板

1. **从模型结果生成结构化数据**（JSON / DataFrame）；
2. **使用模板自动生成自然语言文本**：
   - Jinja2 模板或自定义字符串模板；
   - 针对每个模块设计播报模板，支持中文数值格式化。

### 7.3 语音合成技术选型

- 使用edge_tts

## 8. 技术栈与实现建议

- **数据层**
  - 存储：MySQL / PostgreSQL / CSV / Parquet
  - ETL：`Pandas` + 定时任务（如 Airflow / crontab）
- **建模层**
  - 语言：Python
  - 主要库：
    - 数据处理：`pandas`, `numpy`
    - 可视化：`matplotlib`, `seaborn`
    - 关联规则：`mlxtend.frequent_patterns` 或自行实现 Apriori/FP-Growth
    - 预测：`statsmodels`（ARIMA）、`fbprophet` / `prophet`、`xgboost`、`pytorch`（LSTM）
    - 聚类：`scikit-learn`（KMeans, GMM）
    - TTS：调用在线 API 或使用 `edge-tts`、本地 TTS 模型
- **应用层**
  - 可视化报表：Flask/Django + 前端页面 或 BI 工具（Power BI / Tableau）
  - 语音播放：Web 前端音频控件 / PC 端播放器
