# 实验报告 · 普适性分析模块（三组正则化数据验证）

> 脚本：`code_files_3/`（config_3 + analysis_engine + 5 模块）
> 输入：正则化引擎产物 `output_2/regularized/{order_1, order_2, 销售数据}/dataset.csv` + capability.json
> 输出：`output_3/{数据集}/{figures,csvs,pkls}` + `cross_dataset_summary.csv`

## 1. 目标与方法
将项目1（绑定固定中文字段）的分析能力升级为**只依赖标准 Schema、按 capability + 运行时校验自适应**的通用模块，使同一套分析代码可直接处理任意经正则化的数据，缺字段降级而非失败。

5 个模块：描述性概览 / 顾客画像+分群 / 关联规则(FP-Growth+HUIM) / 个性化推荐(多召回+可靠度校准融合) / 促销因果(朴素 vs DML)。

## 5. 关于推荐的诚实说明
零售推荐存在"热门悖论"：刚需 staples（根茎/花果/叶菜）几乎人人购买，**纯热门推荐同样的 10 件即可命中 44%**，但覆盖率仅 0.0018（给所有人推一样的东西，无个性化价值）。本引擎融合用可靠度校准（图嵌入/热门权重高、类目低）+ 加权和，得 HitRate@10=0.377，略低于纯热门，但**覆盖率 0.024（约 13 倍）**且具个性化与可解释性——与项目1「融合换取覆盖与个性化」的结论一致。order_2（1955 商品、4年）命中普遍低（catalog 大、稀疏），属正常；order_1（单购为主）推荐稀疏。
